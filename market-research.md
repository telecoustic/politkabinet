*Дата: 13 апреля 2026*

## 1. Конкурентный анализ

### 1.1. Прямые конкуренты (политические тесты)

---

#### Political Compass (politicalcompass.org)

**Что делает:** Тест политических взглядов на 2 оси (экономическая лево-право, социальная либертарианство-авторитаризм). Существует с 2001 года. Результат — точка на квадранте.

| Параметр | Значение |
|---|---|
| Количество вопросов | 62 |
| Формат вопросов | Утверждения, шкала согласия (agree/disagree) |
| Оси | 2 (экономическая, социальная) |
| Языки | Английский (есть русский перевод) |
| Трафик | ~1.5 млн визитов/мес (Semrush, январь 2026) |
| Аудитория | 62% мужчины, ядро 18-24, основной трафик — США, UK, Канада |

**Монетизация:**
- Реклама на сайте
- Донаты через PayPal (предлагают 1-5 фунтов)
- Лицензирование графиков — изображения используются в учебниках (Oxford University Press и др.)
- Семинары Political Compass Seminars
- Спонсорство

**Виральная механика:**
- Результат — точка на графике, удобно для скриншота
- Нет встроенного сравнения с друзьями
- Нет кнопки шаринга в мессенджеры
- Виральность за счёт узнаваемости бренда и обсуждений в Reddit/Twitter

**Сильные стороны:**
- Бренд с 25-летней историей, стандарт в категории
- Простая и узнаваемая визуализация (квадрант)
- Сравнение с политиками и партиями
- Бесплатный, без регистрации

**Слабые стороны:**
- Вопросы абстрактные и прямые (не ситуационные) — социально одобряемые ответы
- Только 2 оси — грубое приближение
- Дизайн устаревший (уровень 2005 года)
- Нет механики сравнения с друзьями
- Русская версия — формальный перевод, не адаптация
- Результат — точка без описания типа, нечем делиться кроме скриншота
- 62 вопроса — долго

**Уроки для Политкабинета:**
- Простая визуализация (квадрант) запоминается, но 4 отдельные шкалы дают больше глубины
- Сравнение с известными людьми/политиками — мощная фича для обсуждений (на будущее)
- Бренд строится десятилетиями, но можно обойти за счёт лучшего UX и виральной механики

#### 8values (8values.github.io)

**Что делает:** Open-source политический тест на 4 оси (8 ценностей). Результат — процентные полосы по каждой оси + ближайшая идеология.

| Параметр | Значение |
|---|---|
| Количество вопросов | 70 |
| Формат вопросов | Утверждения, шкала Strongly Agree — Strongly Disagree |
| Оси | 4 (экономика, дипломатия, гражданские свободы, общество) |
| Языки | Английский. Есть русский форк (8values-ru.github.io) |
| Трафик | ~85-112K визитов/мес (Similarweb, 2024) |

**Монетизация:**
- Google AdSense (минимальная)
- Open-source проект, по сути не коммерческий

**Виральная механика:**
- Нет встроенного шаринга
- Нет сравнения с друзьями
- Виральность через Reddit и политические сообщества

**Сильные стороны:**
- 4 оси — более точная модель, чем Political Compass
- Open-source — породил экосистему форков (9axes, SapplyValues, 8values-ru)
- Бесплатный, без регистрации

**Слабые стороны:**
- 70 вопросов — слишком длинно
- Стена текста на результате, не скриншотабельно
- Нет запоминающихся ярлыков/типов — только проценты
- Нет ситуационных вопросов
- Дизайн минималистичный до бедности
- Нет сравнения с друзьями
- Русский форк — буквальный перевод

**Уроки для Политкабинета:**
- Модель 4 осей — та самая, которую мы берём за основу (Политкабинет на ней и построен)
- Отсутствие запоминающихся ярлыков — главная упущенная возможность. Именно здесь Политкабинет отстраивается
- Отсутствие сравнения — ключевой разрыв рынка

#### PolitiScales (politiscales.party)

**Что делает:** Политический тест на 8 осей, вдохновлён 8values. Результат — полосы по осям + генерируемый «флаг».

| Параметр | Значение |
|---|---|
| Количество вопросов | 117 |
| Формат вопросов | Утверждения, 5-балльная шкала согласия |
| Оси | 8 (конструктивизм/эссенциализм, реабилитация/наказание, прогресс/консерватизм, интернационализм/национализм, коммунизм/капитализм, регуляция/laissez-faire, экология/продуктивность, революция/реформа) |
| Языки | Английский, французский (многоязычный форк на GitHub) |

**Монетизация:**
- Отсутствует. Open-source, community-maintained

**Виральная механика:**
- Генерируемый «флаг» из символов — уникальная визуальная фича, хорошо для скриншотов
- Шаринг через URL с закодированными результатами
- Нет прямого сравнения с друзьями

**Сильные стороны:**
- 8 осей — максимальная детализация
- Генерируемый флаг — мемная, шарабельная фича
- URL с результатом (можно кинуть ссылку)

**Слабые стороны:**
- 117 вопросов — неприемлемо долго для массовой аудитории
- Нет ярлыков/типов — только проценты
- Нет сравнения с друзьями
- Вопросы содержат шокирующие утверждения (расизм, гомофобия) — не нейтральный тон
- Нет русской версии

**Уроки для Политкабинета:**
- Генерируемый визуальный артефакт (флаг) — сильная виральная механика. Подумать о визуальном «гербе типа» или подобном на будущее
- URL с закодированным результатом — именно то, что мы делаем (politkabinet.ru/t/85-41-81-55)

#### IDRlabs Political Coordinates Test (idrlabs.com)

**Что делает:** Тест на 2 оси, часть платформы IDRlabs (сотни психологических и политических тестов). Академический подход.

| Параметр | Значение |
|---|---|
| Количество вопросов | 36 |
| Формат вопросов | Утверждения, agree/disagree |
| Оси | 2 (экономическая лево-право, социальная) |
| Языки | Много, включая русский |

**Монетизация:**
- Тяжёлая рекламная модель: AdThrive, Google Tag Manager, Facebook Pixel
- Тесты бесплатные, доход от рекламы и (предположительно) агрегированных данных
- Академическая рецензия (Dr. Kellen Gracey, Ph.D.) — добавляет авторитетности

**Виральная механика:**
- Нет встроенного сравнения
- Результат — точка на квадранте, можно сделать скриншот

**Сильные стороны:**
- Короткий (36 вопросов)
- Академическая валидация
- Часть большой платформы — кросс-промоушен между тестами
- Хороший русский перевод

**Слабые стороны:**
- Только 2 оси
- Агрессивная реклама (множество трекеров)
- Нет ярлыков/типов
- Нет сравнения с друзьями
- Визуально сухой

**Уроки для Политкабинета:**
- 36 вопросов — ближе к нашему формату (24). Короткий тест — сильное конкурентное преимущество
- Кросс-промоушен между тестами — на будущее, если делать серию тестов

#### 16personalities (16personalities.com)

**Что делает:** Тест типа MBTI (16 типов личности). Золотой стандарт виральных тестов личности.

| Параметр | Значение |
|---|---|
| Прохождений | >1 млрд (все время) |
| Трафик | ~17.1 млн визитов/мес (Similarweb, июнь 2025) |
| Языки | 45+ |
| Формат | Утверждения, шкала согласия |
| Компания | NERIS Analytics Limited, Кембридж, UK |

**Монетизация (freemium):**
- Бесплатно: тест + базовое описание типа
- Платно: Premium Profile ($29-49), Career Suite ($29), Pro Suite
- Профессиональные отчёты для организаций
- Купоны и скидочные акции

**Виральная механика:**
- Запоминающиеся типы-ярлыки (Architect, Mediator, Commander)
- Подробные описания «это про меня»
- Результат удобен для скриншота
- Типы стали частью поп-культуры (в Tinder-профилях, резюме)
- Нет прямого сравнения с друзьями (!)

**Сильные стороны:**
- Золотой стандарт виральности — proof of concept для нашей модели
- Отличный UX и дизайн
- Описания типов — глубокие и узнаваемые
- Freemium работает: даже при минимальной конверсии, 17M трафика = серьёзный доход

**Слабые стороны:**
- Нет сравнения с друзьями (только через внешние сервисы)
- Тема — личность в целом, не политика
- Академически спорный (MBTI критикуют психологи)

**Уроки для Политкабинета:**
- **Главный урок**: запоминающиеся ярлыки типов + узнаваемые описания = виральность. Именно это, а не точность модели, делает тест вирусным
- Freemium с порогом ценности: бесплатное должно быть достаточно хорошим, чтобы расшарить; платное — достаточно глубоким, чтобы захотеть
- 16personalities не имеет сравнения с друзьями — и это их упущение, которое мы заполняем

#### Rice Purity Test

**Что делает:** 100 yes/no вопросов о жизненном опыте. Результат — одно число (0-100). Чем ниже, тем «грешнее».

| Параметр | Значение |
|---|---|
| Вопросов | 100 |
| Формат | Чеклист yes/no |
| Результат | Одно число (score) |
| Хэштег | #RicePurityTest — миллиарды просмотров в TikTok |

**Монетизация:** Рекламные клоны (оригинал — с 1924 года, Rice University). Десятки сайтов-копий с рекламой.

**Виральная механика (выдающаяся):**
- Результат — одно число. Максимально просто для шаринга
- Естественное сравнение: «у меня 47, а у тебя?»
- Тема (сексуальный и жизненный опыт) провоцирует обсуждение
- TikTok-тренд: снимают реакцию на свой score
- Друзья дарят/челленджат друг друга

**Уроки для Политкабинета:**
- Одно число или короткий ярлык — максимум виральности
- Провокационная тема = обсуждения = шаринг
- Простота результата > детализация результата (для виральности)

---

#### Human Benchmark (humanbenchmark.com)

**Что делает:** Набор когнитивных тестов (время реакции, память, визуальная память и др.). 8 тестов, каждый с результатом и перцентилем.

**Монетизация:** Бесплатный. Реклама минимальная.

**Виральная механика:**
- Результат — число + перцентиль (топ X%)
- Соревновательный элемент: «я в топ 5%, а ты?»
- Повторяемость: можно улучшать свой результат

**Уроки для Политкабинета:**
- Перцентиль и рейтинг — сильный триггер для шаринга (на будущее, когда будут данные)
- Повторяемость не релевантна для политического теста (взгляды не «тренируются»), но динамический профиль (вариант B из pricing-strategy) — это аналог

#### Тесты в соцсетях (BuzzFeed, uQuiz, TikTok-квизы)

**Что делает:** Короткие развлекательные тесты («Какой ты персонаж?», «Какой город тебе подходит?»).

**Монетизация:** Реклама (BuzzFeed), lead generation, спонсорский контент.

**Виральная механика:**
- Короткие (5-10 вопросов)
- Результат = картинка для Stories
- Tag-a-friend (+30-50% engagement)
- Персонализация: «это про меня!»

**Уроки для Политкабинета:**
- Формат результата для Stories/скриншота — обязателен
- Tag-a-friend механика: «кинь другу, пусть тоже пройдёт» — встроена в нашу модель через сравнение
- Квизы с неожиданными результатами шарятся лучше

### 2.3. Уникальная комбинация (почему не копируется одной фичей)

Политкабинет — единственный продукт, который одновременно:

1. **Ситуационные вопросы** — не абстрактные утверждения, а бытовые ситуации. Снижает социальную желательность ответов, делает процесс интересным
2. **16 типов с ироничными ярлыками** — запоминающиеся, мемные, провоцирующие обсуждение
3. **Встроенное сравнение с друзьями** — одно действие: вставил код, увидел результат. Ни один политический тест этого не имеет
4. **24 вопроса (~7-10 минут)** — самый короткий среди серьёзных (Political Compass: 62, 8values: 70, PolitiScales: 117)
5. **Нативный русский** — не перевод, а продукт на русском с нуля
6. **Ироничный тон** — не академический и не снисходительный

Любой конкурент может скопировать одну фичу. Комбинацию всех шести — нет, потому что это другой продуктовый подход (развлечение + самопознание + социальное взаимодействие).

## 3. Тренды рынка

### 3.1. Тренды в онлайн-квизах / тестах (2025-2026)

**1. Короткие форматы побеждают**
Внимание сокращается. Тесты на 5-15 минут имеют значительно выше completion rate, чем на 20+. Наши 24 вопроса (~7-10 минут) — в зоне оптимума.

**2. TikTok как поисковик для квизов**
Gen Z ищет тесты через TikTok, не через Google. Хэштеги #personalitytest, #politicalcompass имеют миллиарды просмотров. Формат: снять видео «прохожу тест» или «реакция на результат».

**3. Визуальные результаты для Stories**
Результат теста должен быть скриншотабельным. Карточка результата — это контент для Instagram Stories, Telegram и TikTok. Политкабинет: экран результата с типом + 4 шкалы — готовый скриншот.

**4. Сравнение и соревнование**
Тренд на friendship quizzes (friendshipquiz2023.com и аналоги). Механика: «создай квиз про себя, кинь другу, посмотри сколько он угадал». Tag-a-friend механика даёт +30-50% engagement.

**5. AI-персонализация**
Тесты начинают использовать AI для генерации персонализированных описаний. 16personalities пока этого не делает — но рынок движется туда. Наш план на итерацию 2 (AI-разбор) — в тренде.

**6. Гибридная монетизация**
35% приложений в 2025 используют гибридную модель (подписки + разовые покупки + реклама). Чистая реклама теряет эффективность. Freemium с upsell на premium-контент — доминирующая модель.

**7. Сезонность**
Интерес к квизам выше осенью-зимой, ниже летом (июнь-июль — провал). Выборы и политические события дают всплески трафика для политических тестов. Запуск осенью — оптимален.

### 3.2. Примеры виральных тестов последних лет

| Тест | Год виральности | Механика | Масштаб |
|---|---|---|---|
| 16personalities | 2015-... (постоянно) | Запоминающиеся типы, описания «про меня» | >1 млрд прохождений |
| Rice Purity Test | 2020 (TikTok-ревайвал) | Одно число, провокационная тема | Миллиарды просмотров хэштега |
| Political Compass | Всплески при выборах | Квадрант, сравнение с политиками | ~1.5 млн/мес |
| Human Benchmark | 2021-2023 | Соревновательный рейтинг, перцентили | Миллионы пользователей |
| «Какой ты...» (BuzzFeed) | 2014-2018 | Короткий, картинка-результат, tag-a-friend | Сотни миллионов прохождений |
| Friendship Quiz | 2023-2024 | Создай квиз → кинь другу → сравни | Десятки миллионов |
| «Same Quiz» (TikTok) | 2025 | AI-powered, персонализация | Тренд в TikTok |

**Закономерность:** Все виральные тесты имеют минимум одну из трёх механик:
1. Запоминающийся результат-ярлык (16personalities)
2. Числовой score для сравнения (Rice Purity, Human Benchmark)
3. Социальное взаимодействие (Friendship Quiz, tag-a-friend)

Политкабинет комбинирует все три: ярлык типа + числовые шкалы + сравнение с другом.

### 3.3. Монетизация политических тестов в мире

| Модель | Примеры | Доход |
|---|---|---|
| Реклама (display ads) | Political Compass, IDRlabs, 8values | Низкий. При 1.5 млн визитов/мес и CPM $2-5 = $3K-7.5K/мес |
| Донаты | Political Compass | Символический. Не бизнес-модель |
| Лицензирование контента | Political Compass (учебники) | Разовые сделки, непредсказуемо |
| Freemium (premium-контент) | 16personalities (не политический, но аналогия) | При 17M трафика и 1% конверсии по $30 = $5.1M/мес. Реальная цифра неизвестна, но масштаб значительный |
| Данные (агрегированные) | IDRlabs (предположительно) | Неизвестно |

**Вывод:** Реклама на политическом тесте — потолок $5-10K/мес при серьёзном трафике. Freemium с контентным upsell — единственная модель, дающая значимый доход. Это подтверждает стратегию из pricing-strategy.md: запускать бесплатно, монетизировать через групповой продукт или динамический профиль.

## 4. Скрипты аналитики (заготовки для SQL)

### 4.1. Конверсия по шагам воронки

```sql
-- Воронка прохождения теста
-- Предпосылка: таблица events (user_id, event_type, created_at, metadata)

SELECT
  step,
  users_count,
  ROUND(users_count * 100.0 / FIRST_VALUE(users_count) OVER (ORDER BY step_order), 1) AS pct_from_start,
  ROUND(users_count * 100.0 / LAG(users_count) OVER (ORDER BY step_order), 1) AS pct_from_prev_step
FROM (
  SELECT 1 AS step_order, 'landing_view' AS step,
    COUNT(DISTINCT user_id) AS users_count
  FROM events WHERE event_type = 'landing_view'
  UNION ALL
  SELECT 2, 'age_entered',
    COUNT(DISTINCT user_id)
  FROM events WHERE event_type = 'age_entered'
  UNION ALL
  SELECT 3, 'name_entered',
    COUNT(DISTINCT user_id)
  FROM events WHERE event_type = 'name_entered'
  UNION ALL
  SELECT 4, 'test_started',
    COUNT(DISTINCT user_id)
  FROM events WHERE event_type = 'test_started'
  UNION ALL
  SELECT 5, 'test_completed',
    COUNT(DISTINCT user_id)
  FROM events WHERE event_type = 'test_completed'
  UNION ALL
  SELECT 6, 'share_clicked',
    COUNT(DISTINCT user_id)
  FROM events WHERE event_type = 'share_clicked'
  UNION ALL
  SELECT 7, 'compare_used',
    COUNT(DISTINCT user_id)
  FROM events WHERE event_type = 'compare_used'
) funnel
ORDER BY step_order;
```

### 4.2. Виральный коэффициент (K-фактор)

```sql
-- K-фактор = (среднее число приглашений на пользователя) * (конверсия приглашённых)
-- K > 1 = экспоненциальный рост, K > 0.5 = здоровый виральный рост

WITH sharers AS (
  -- Пользователи, которые поделились ссылкой
  SELECT
    user_id,
    COUNT(*) AS shares_count
  FROM events
  WHERE event_type = 'share_clicked'
  GROUP BY user_id
),
referral_visits AS (
  -- Визиты по реферальным ссылкам (с кодом друга в URL)
  SELECT
    COUNT(DISTINCT user_id) AS referred_visitors
  FROM events
  WHERE event_type = 'landing_view'
    AND metadata->>'referrer_code' IS NOT NULL
),
referral_completions AS (
  -- Из них — прошли тест до конца
  SELECT
    COUNT(DISTINCT e.user_id) AS referred_completed
  FROM events e
  JOIN events e2 ON e.user_id = e2.user_id
  WHERE e.event_type = 'test_completed'
    AND e2.event_type = 'landing_view'
    AND e2.metadata->>'referrer_code' IS NOT NULL
),
total_completions AS (
  SELECT COUNT(DISTINCT user_id) AS total_users
  FROM events WHERE event_type = 'test_completed'
)
SELECT
  -- Среднее число шаров на прошедшего
  ROUND(
    (SELECT SUM(shares_count) FROM sharers)::numeric
    / NULLIF((SELECT total_users FROM total_completions), 0),
    2
  ) AS avg_shares_per_user,

  -- Конверсия реферальных визитов в прохождения
  ROUND(
    (SELECT referred_completed FROM referral_completions)::numeric
    / NULLIF((SELECT referred_visitors FROM referral_visits), 0),
    2
  ) AS referral_conversion,

  -- K-фактор
  ROUND(
    (SELECT SUM(shares_count) FROM sharers)::numeric
    / NULLIF((SELECT total_users FROM total_completions), 0)
    * (SELECT referred_completed FROM referral_completions)::numeric
    / NULLIF((SELECT referred_visitors FROM referral_visits), 0),
    2
  ) AS k_factor;
```

### 4.3. Время прохождения

```sql
-- Среднее и медианное время прохождения теста
-- + распределение по бакетам

WITH durations AS (
  SELECT
    e_start.user_id,
    EXTRACT(EPOCH FROM (e_end.created_at - e_start.created_at)) / 60.0 AS duration_minutes
  FROM events e_start
  JOIN events e_end ON e_start.user_id = e_end.user_id
  WHERE e_start.event_type = 'test_started'
    AND e_end.event_type = 'test_completed'
)
SELECT
  COUNT(*) AS total_completions,
  ROUND(AVG(duration_minutes), 1) AS avg_minutes,
  ROUND(PERCENTILE_CONT(0.5) WITHIN GROUP (ORDER BY duration_minutes), 1) AS median_minutes,
  ROUND(PERCENTILE_CONT(0.25) WITHIN GROUP (ORDER BY duration_minutes), 1) AS p25_minutes,
  ROUND(PERCENTILE_CONT(0.75) WITHIN GROUP (ORDER BY duration_minutes), 1) AS p75_minutes
FROM durations
WHERE duration_minutes BETWEEN 1 AND 60; -- отсекаем аномалии

-- Распределение по бакетам
SELECT
  CASE
    WHEN duration_minutes < 3 THEN '< 3 мин (спидранеры)'
    WHEN duration_minutes < 5 THEN '3-5 мин (быстрые)'
    WHEN duration_minutes < 8 THEN '5-8 мин (нормальные)'
    WHEN duration_minutes < 12 THEN '8-12 мин (вдумчивые)'
    WHEN duration_minutes < 20 THEN '12-20 мин (медленные)'
    ELSE '> 20 мин (отвлекались)'
  END AS bucket,
  COUNT(*) AS users_count,
  ROUND(COUNT(*) * 100.0 / SUM(COUNT(*)) OVER (), 1) AS pct
FROM durations
WHERE duration_minutes BETWEEN 1 AND 60
GROUP BY 1
ORDER BY MIN(duration_minutes);
```

### 4.4. Распределение типов

```sql
-- Распределение 16 типов среди прошедших
-- Проверка: если распределение слишком неравномерное, вопросы требуют калибровки

WITH results AS (
  SELECT
    user_id,
    metadata->>'type_name' AS type_name,
    (metadata->>'economy')::int AS economy,
    (metadata->>'diplomacy')::int AS diplomacy,
    (metadata->>'government')::int AS government,
    (metadata->>'society')::int AS society
  FROM events
  WHERE event_type = 'test_completed'
)
SELECT
  type_name,
  COUNT(*) AS users_count,
  ROUND(COUNT(*) * 100.0 / SUM(COUNT(*)) OVER (), 1) AS pct,
  -- Ожидание при равном распределении: 6.25% на тип
  ROUND(COUNT(*) * 100.0 / SUM(COUNT(*)) OVER () - 6.25, 1) AS deviation_from_expected
FROM results
GROUP BY type_name
ORDER BY users_count DESC;

-- Средние значения по осям (для проверки калибровки)
SELECT
  ROUND(AVG(economy), 1) AS avg_economy,
  ROUND(AVG(diplomacy), 1) AS avg_diplomacy,
  ROUND(AVG(government), 1) AS avg_government,
  ROUND(AVG(society), 1) AS avg_society,
  -- В идеале средние близки к 50
  ROUND(STDDEV(economy), 1) AS stddev_economy,
  ROUND(STDDEV(diplomacy), 1) AS stddev_diplomacy,
  ROUND(STDDEV(government), 1) AS stddev_government,
  ROUND(STDDEV(society), 1) AS stddev_society
FROM results;
```

### 4.5. Точки отвала

```sql
-- На каком вопросе бросают тест
-- Предпосылка: событие question_answered с номером вопроса

WITH last_question AS (
  SELECT
    user_id,
    MAX((metadata->>'question_number')::int) AS last_q
  FROM events
  WHERE event_type = 'question_answered'
  GROUP BY user_id
),
dropoffs AS (
  SELECT
    lq.user_id,
    lq.last_q,
    CASE WHEN tc.user_id IS NOT NULL THEN false ELSE true END AS dropped
  FROM last_question lq
  LEFT JOIN events tc ON lq.user_id = tc.user_id AND tc.event_type = 'test_completed'
)
SELECT
  last_q AS dropped_after_question,
  COUNT(*) AS users_dropped,
  ROUND(COUNT(*) * 100.0 / SUM(COUNT(*)) OVER (), 1) AS pct_of_all_dropoffs
FROM dropoffs
WHERE dropped = true
GROUP BY last_q
ORDER BY last_q;

-- Общая воронка: retention по вопросам
SELECT
  q_num,
  COUNT(DISTINCT user_id) AS users_reached,
  ROUND(
    COUNT(DISTINCT user_id) * 100.0
    / FIRST_VALUE(COUNT(DISTINCT user_id)) OVER (ORDER BY q_num),
    1
  ) AS retention_pct
FROM (
  SELECT
    user_id,
    (metadata->>'question_number')::int AS q_num
  FROM events
  WHERE event_type = 'question_answered'
) q
GROUP BY q_num
ORDER BY q_num;
```

### 4.6. Дашборд: ежедневные ключевые метрики

```sql
-- Ежедневная сводка для мониторинга после запуска

SELECT
  d.date,
  COALESCE(v.visits, 0) AS visits,
  COALESCE(s.starts, 0) AS test_starts,
  COALESCE(c.completions, 0) AS test_completions,
  COALESCE(sh.shares, 0) AS shares,
  COALESCE(cmp.compares, 0) AS comparisons,
  -- Конверсии
  ROUND(COALESCE(s.starts, 0) * 100.0 / NULLIF(v.visits, 0), 1) AS visit_to_start_pct,
  ROUND(COALESCE(c.completions, 0) * 100.0 / NULLIF(s.starts, 0), 1) AS start_to_complete_pct,
  ROUND(COALESCE(sh.shares, 0) * 100.0 / NULLIF(c.completions, 0), 1) AS complete_to_share_pct,
  -- Реферальный трафик
  COALESCE(r.referral_visits, 0) AS referral_visits,
  ROUND(COALESCE(r.referral_visits, 0) * 100.0 / NULLIF(v.visits, 0), 1) AS referral_pct
FROM generate_series(
  CURRENT_DATE - INTERVAL '30 days',
  CURRENT_DATE,
  INTERVAL '1 day'
) AS d(date)
LEFT JOIN (
  SELECT DATE(created_at) AS date, COUNT(DISTINCT user_id) AS visits
  FROM events WHERE event_type = 'landing_view' GROUP BY 1
) v ON d.date = v.date
LEFT JOIN (
  SELECT DATE(created_at) AS date, COUNT(DISTINCT user_id) AS starts
  FROM events WHERE event_type = 'test_started' GROUP BY 1
) s ON d.date = s.date
LEFT JOIN (
  SELECT DATE(created_at) AS date, COUNT(DISTINCT user_id) AS completions
  FROM events WHERE event_type = 'test_completed' GROUP BY 1
) c ON d.date = c.date
LEFT JOIN (
  SELECT DATE(created_at) AS date, COUNT(DISTINCT user_id) AS shares
  FROM events WHERE event_type = 'share_clicked' GROUP BY 1
) sh ON d.date = sh.date
LEFT JOIN (
  SELECT DATE(created_at) AS date, COUNT(DISTINCT user_id) AS compares
  FROM events WHERE event_type = 'compare_used' GROUP BY 1
) cmp ON d.date = cmp.date
LEFT JOIN (
  SELECT DATE(created_at) AS date, COUNT(DISTINCT user_id) AS referral_visits
  FROM events WHERE event_type = 'landing_view'
    AND metadata->>'referrer_code' IS NOT NULL GROUP BY 1
) r ON d.date = r.date
ORDER BY d.date DESC;
```

## 5. Ключевые выводы

### Рынок

1. **Политические тесты — зрелая категория с устаревшими продуктами.** Political Compass (2001), 8values (2017), PolitiScales (2018) — все делались программистами/политологами для себя, без продуктового мышления. Ни один не обновлял UX за последние 5 лет.

2. **Русскоязычный рынок пуст.** Все русские варианты — переводы или минимальные клоны. Нет ни одного продукта, сделанного с нуля для русскоязычной аудитории с учётом виральных механик.

3. **Сравнение с друзьями — ни у кого нет.** Это главный разрыв рынка. Все тесты — solo experience. Социальное взаимодействие ограничивается «сделай скриншот и кинь в чат».

4. **Ситуационные вопросы — ни у кого нет.** Все используют абстрактные утверждения. Ситуационный формат — уникальное позиционирование.

### Риски

1. **Трафик.** Политические тесты живут на органическом трафике. Political Compass — 1.5 млн/мес при 25-летнем бренде. Новый продукт должен набрать виральный рост с нуля.

2. **Монетизация.** Реклама на квизах даёт копейки. Freemium требует масштаба. При <50K прохождений/мес платная версия не имеет смысла — сначала нужно доказать product-market fit бесплатной.

3. **Чувствительность темы.** Политика в России — токсичная тема. Ироничный тон снижает градус, но блогеры могут бояться рекомендовать. Ситуационный формат (без прямых политических вопросов) — правильная стратегия снижения этого риска.

### Рекомендации

1. **Запускать как есть** — бесплатная версия с виральной механикой. Не тратить время на платные фичи до подтверждения виральности.

2. **Приоритет 1 — механика шаринга.** Кнопка «Поделиться» + OG-теги для красивого превью в Telegram — критичнее любой другой фичи.

3. **Приоритет 2 — скриншотабельный результат.** Карточка типа должна выглядеть как готовый пост для Stories.

4. **Приоритет 3 — ивент-трекинг.** Без данных невозможно принимать решения. Простые события (GA4 или Posthog free tier) с первого дня.

5. **Целевые метрики первого месяца:**

| Метрика | Целевой порог |
|---|---|
| Визит → прохождение | > 50% |
| Прохождение → шаринг | > 15% |
| Друг по ссылке → прохождение | > 40% |
| K-фактор | > 0.5 |
| Медианное время прохождения | 7-12 мин |
